Definitions

Blockcore Vault is built upon some of the newest in decentralized technology and blockchain related software, making it use of new terminology that has not yet been widely adopted or settled.

Some of the terminology within Blockcore Vault can change, depending on feedback from users, community and the standard groups.

Vocabulary

Vault: The product name for our offering that supports multiple open standards in the same software. Vault implements various open standards to different degrees. This normally referes to a single deployed instance of the Blockcore Vault software.

Verifiable Data Registry: This is what the Blockcore Vault software is.

Identity Vault: A specialized Vault (see below) that holds DID Documents (see below). The documents stored in the Identity Vault can be retrieved using REST APIs and using DID Resolvers.

Data Vault: Single instance of an data vault that is hosted on a Blockcore Vault instance. The data vault is created by submitting an data vault configuration to the Blockcore Vault. The data vault configuration contains setup for replication, synchronization, conflict resolution, access control and more.

Vault Operator: Any entity, including individuals, who runs an Blockcore Vault Instance on a device or infrastructure they control.

Vault User: An entity that controls the private key to control an DID and transmits messages via an Vault, which may be running on a device in their possession, or on the device or infrastructure of a Vault Operator.

Decentralized Identifiers (DID): Unique ID URI string and PKI metadata document format for describing the cryptographic keys and other fundamental PKI values linked to a unique, user-controlled, self-sovereign identifier in a target system (e.g., blockchain, distributed ledger). https://w3c.github.io/did-core/.

DID Document: A document that describes keys and other information about an DID Subject.

DID Resolver: A library / code that allows lookup of DID Documents against an "DID Registry"/"DID Hub". Learn more at https://github.com/decentralized-identity/universal-resolver.

Verifiable Credential (VC): Verifiable Credentials, a signed JSON-document that contains link to specific DID that performed the signing.

Identity Profile (Identity Hub Profile): This is a signed VC with metadata regarding a specific DID. Can describe information about an Vault instance, computer, serverm service, business or corporate entity or individual. See example: https://identity.foundation/identity-hub/spec/#object-format.

Well Known DID Configuration: A document that exists on an known URL on a running Vault and returns information regarding the DID of a specific instance. https://identity.foundation/specs/did-configuration/.

Well Known Vault Configuration: A document that exists on an known URL on a running Vault and returns information regarding the Vault instance. This is Blockcore-specific and not based on an open standard.

Definitions not used

Identity Hub: "A decentralized personal and application data storage and message relay node, as defined in the DIF Identity Hub specification." https://identity.foundation/identity-hub/spec/.

Confidential Storage (formally Encrypted Data Vault): This specification is partly implemented by Blockcore Vault. It is used to store and retrieve VCs. https://identity.foundation/confidential-storage/.
